                                                                        Yann TM Yann TM • 4 years ago

                                                                        Or actually looking more closely at the diagrams there is some confusion still for me between part and component.

                                                                        The component is a definition of a type (like a class), but the text says the connectors should
                                                                        ""
                                                                        It specifies a link that enables communication between two or more instances.
                                                                        ""
                                                                        So isn't it incorrect to link components directly and not instances thereof using connectors ?

                                                                        In other words the diagrams that show these connectors (ALL the diagrams of the section titled "Connectors") are subtly wrong, the boxes (which are component instances) should have a "roleName:Type" instead of just a type.

                                                                        And these diagrams should probably be enclosed in some structured classifier (typically a component with <<subsystem>> stereotype) or at least a frame as a composite structure diagram. 

                                                                        Edit: After checking this is indeed required by my UML 2 tool (rational soft' architect v8) before it will allow me to draw those diagrams and set connectors between instances. I cannot directly connect components using connectors, I need to make a composite structure diagram (and instantiate the components) to get that feature.
